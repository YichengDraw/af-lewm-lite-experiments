\section*{Setup}
The study uses the official \texttt{pusht\_expert\_train.h5} extracted from \texttt{pusht\_expert\_train.h5.zst}. The active file is \textbf{46.30 GB} with \textbf{2,336,736} cached PushT samples. All primary models and follow-up sweep variants were trained on the same RTX 4070 Laptop GPU with the same fixed budget:

\begin{itemize}[leftmargin=1.2em, itemsep=0.2em, topsep=0.2em]
  \item 10 epochs
  \item 200 training batches per epoch
  \item 20 validation batches per epoch
  \item batch size 4
  \item bf16 training
  \item final 50-episode planning evaluation at epoch 10
\end{itemize}

This budget gives a fair relative comparison on the official dataset without turning the experiment into an impractically long full-data sweep.

The parameter differences are small. AF-LeWM-lite v1 adds \textbf{0.74\%} over baseline, and v2 adds \textbf{1.12\%}. The planning ranking therefore is not explained by a large capacity jump.

The \textbf{core loss} is the shared JEPA objective,
\[
\mathcal{L}_{\mathrm{core}} = \mathcal{L}_{\mathrm{pred}} + 0.09\,\mathcal{L}_{\mathrm{sigreg}},
\]
which is directly comparable across baseline, v1, and v2. The larger \texttt{validate/loss\_epoch} used during v2 training is not comparable because it also includes the AF-specific nuisance regularizers.

\section*{Interpretation}
\textbf{AF-LeWM-lite v1 is not a good stopping point.} The split plus invariance and independence losses alone degrade both latent prediction quality and planning success on official PushT. This matches the earlier concern that \texttt{app\_emb} lacked a strong positive job.

\textbf{The dynamics nuisance adversary is the key mechanism.} Removing it restores the shared core loss almost exactly to baseline, then collapses control to \textbf{0/50}. Weakening the adversarial pressure also pulls the core loss back down and reduces success to \textbf{1/50}. That means the adversary is not incidental regularization. It is the main source of the current control gain.

\textbf{The tradeoff is real and currently sharp.} Original v2 reaches the best control result while showing the worst shared core loss by a large margin. The follow-up sweep shows that static weight reduction does not solve this cleanly. Better shared modeling and better PushT control are still pulling in different directions under the current setup.

\textbf{The control edge is fragile.} The second 50-episode evaluation seed reduces the primary comparison to \textbf{5/100} for v2 versus \textbf{4/100} for baseline. That keeps v2 slightly ahead and removes any basis for a strong robustness claim.

\section*{Conclusion}
The current conclusion is specific and actionable:

\begin{itemize}[leftmargin=1.2em, itemsep=0.2em, topsep=0.2em]
  \item Keep the \textbf{original AF-LeWM-lite v2} mechanism as the active branch.
  \item Drop \textbf{AF-LeWM-lite v1} as a candidate baseline for future reporting.
  \item Stop prioritizing static weight reduction. The follow-up sweep shows that weaker adversarial pressure improves the shared core loss and weakens control.
  \item Run the next experiment on \textbf{adversary scheduling}: delayed onset or gradual ramp of the dynamics nuisance adversary.
  \item Treat the current result as a \textbf{matched-compute PushT tradeoff study}, not a strong absolute PushT benchmark. All success rates remain low under this short budget.
  \item Use \textbf{both} shared core loss and success rate in future comparisons. Success alone is too noisy at 50 episodes, and total training loss is not cross-model comparable once AF-specific regularizers are active.
  \item Report multi-seed aggregates before claiming that AF-LeWM-lite improves PushT planning.
\end{itemize}

In one sentence: \textbf{the factorized-latent idea is still viable, the original v2 objective has a fragile control edge, and the highest-value next step is adversary schedule tuning rather than static loss reweighting or more model complexity.}
