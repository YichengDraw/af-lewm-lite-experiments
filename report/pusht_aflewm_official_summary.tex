\section*{Implementation Fixes}
The rerun used a corrected pipeline in both training and evaluation:

\begin{itemize}[leftmargin=1.2em, itemsep=0.2em, topsep=0.2em]
  \item \texttt{JEPA.rollout} now keeps the real past action blocks in context and appends candidate future blocks one step at a time. The previous logic could overwrite history slots with future candidate actions.
  \item \texttt{eval.py} now reconstructs the raw dataset history window and compresses it into the model-scale history expected by the predictor. The previous path evaluated multi-frame models with an effectively broken history signal.
  \item Goal indexing now includes the true goal frame, and valid start-step sampling no longer drops the final valid index.
  \item Evaluation result files are overwritten, not appended, and the report parser reads the latest metric entry.
  \item AF auxiliary passes now freeze projector BatchNorm statistics during augmented forwards so the main dynamics branch does not receive augmentation-induced BN drift.
\end{itemize}

These fixes were exercised by rerunning the formal training jobs and all reported evaluations.

\section*{Setup}
The study uses the official \texttt{pusht\_expert\_train.h5} with \textbf{2,336,736} cached samples. All three primary models were trained on the same RTX 4070 Laptop GPU under the same short budget:

\begin{itemize}[leftmargin=1.2em, itemsep=0.2em, topsep=0.2em]
  \item 10 epochs
  \item 200 training batches per epoch
  \item 20 validation batches per epoch
  \item batch size 4
  \item bf16 training
  \item 50-episode planning evaluation at epoch 10
  \item second 50-episode evaluation with \texttt{seed=43}
\end{itemize}

The fair cross-model loss is the shared JEPA objective
\[
\mathcal{L}_{\mathrm{core}} = \mathcal{L}_{\mathrm{pred}} + 0.09\,\mathcal{L}_{\mathrm{sigreg}}.
\]
\texttt{validate/loss\_epoch} is not directly comparable once AF-specific nuisance losses are active, especially for v2.

\section*{Interpretation}
\textbf{The corrected pipeline changes the conclusion.} The old report overstated v2. After the history, goal-step, result-writing, and auxiliary BatchNorm fixes, v2 no longer has the best aggregate PushT result.

\textbf{v1 is the most credible current variant.} It is the only AF model that improved planning success on both evaluation seeds. The gain is small, but it is also the most consistent signal in the corrected study.

\textbf{Both AF variants worsen the shared JEPA objective.} Baseline still gives the best core loss, v1 is slightly worse, and v2 is worse again. The current factorization does not improve representation learning quality on the shared predictive objective.

\textbf{v2 adds complexity without a reliable control gain.} Its extra nuisance heads and adversarial branch increase training time and degrade the fair loss comparison, while the two-seed control result remains weaker than v1.

\textbf{The remaining weakness is methodological, not an implementation artifact.} The corrected code was exercised through formal reruns of all three training jobs and six 50-episode evaluations. Success rates remain low after those fixes, so the current budgeted setup is genuinely weak on official PushT.

\section*{Conclusion}
The corrected recommendation is straightforward:

\begin{itemize}[leftmargin=1.2em, itemsep=0.2em, topsep=0.2em]
  \item Treat \textbf{AF-LeWM-lite v1} as the current best branch under the short PushT budget.
  \item Stop using the earlier single-seed v2 result as evidence that the nuisance-adversarial extension is superior.
  \item Compare models with \textbf{shared core loss} and \textbf{multi-seed success aggregates}. Total training loss is not a fair cross-model metric once AF-specific losses are active.
  \item Use the next round for a stronger positive appearance objective or a different training schedule, not for larger v2 complexity on the same short budget.
  \item Report the current result as a \textbf{small exploratory gain with high uncertainty}, not as a benchmark win.
\end{itemize}

In one sentence: \textbf{after implementation fixes and full reruns, AF-LeWM-lite v1 is the best current variant on official PushT, but all three models remain far too weak for a strong claim.}
